What is cost per install?

CPI is what you pay for each app install from a paid campaign: CPI = ad spend ÷ installs. It's the headline metric for mobile app user acquisition — but installs alone don't pay the bills, so this tool also carries installs through to paying users and revenue.

CPI vs cost per paying user

A cheap CPI is meaningless if those users never convert. The number that matters is cost per paying user (spend ÷ paying users) measured against what a paying user is worth. Optimize campaigns toward paying users and revenue, not raw install volume.

Typical CPI ranges

CPI varies widely by platform, geography and vertical — often anywhere from under $1 in low-cost regions to $5+ for competitive categories on iOS in tier-1 countries. Benchmark against your own funnel: the target is a CPI (and downstream cost per paying user) your app economics can support.

Frequently asked questions

How do I calculate cost per install?
Divide total ad spend by the number of installs generated. $5,000 spend for 2,500 installs is a $2.00 CPI.
What is a good CPI?
It depends on platform, region and how much a user is worth to you. A good CPI is one where downstream revenue per user comfortably exceeds acquisition cost.
Why track paying users, not just installs?
Cheap installs that never convert lose money. Cost per paying user against user value is the metric that determines profitability.
